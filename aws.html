Découvrez le cloud avec Amazon Web Services

J’ai fait cette activité car je voulais toucher à d’autres Cloud que Google drive. J’avais déjà utilisé le SAAS et le IAAS lors du travail de Madame Van den Schriek. Je voulais également tester le PAAS et j’ai donc suivi cette formation.

En premier lieu, j’ai eu un rappel sur le Cloud, l’avantage de celui-ci et les différents types existants. Le plus gros avantage du Cloud est que l’utilisateur évite tous les problèmes physiques d’un serveur et laisse la société s’occuper de cela. Le serveur utilisé n’est pas fixe, l’utilisateur loue une quantité de puissance de calcul et de stockage.

Ensuite j’ai découvert quelques services. Amazon donne accès à énormément de services tels que EC2, RDS et S3 et bien d’autres que j’aborderai plus loin. Pour éviter quelques complications, Amazon a également mis en place le système Elastic Beanstalk qui permet de faciliter la mise en place des autres services.

EC2 : Elastic Compute Cloud. Service originel d’Amazon et le plus utilisé chez AWS. Ce service permet d’héberger un site web classique. Il est possible de mettre en place le système d’adresse élastique, ce qui permet de garder une adresse statique pour le serveur utilisé. Un système de clés, facilite la connexion à son serveur de manière sécurisée. L’équilibrage de charge permet de rediriger les clients si une congestion arrive. L’auto-scaling, permet d’ajouter des serveurs si le trafic augmente.

RDS : Relational Database Service. Gestion de la base de données. RDS facilite grandement la mise en place d’une base de données pour notre site web. Tout d’abord, la base de données est déjà installée, pas besoin de faire des mises à jour, sauvegarde et restauration simple et possibilité d’augmenter le nombre de serveurs si besoin. En quelques clics, il est possible d’installer plusieurs moteurs de base de données (MySQL, Postgresql, Mariadb, …). Bien entendu, une fois la base de données en place, il faut la faire communiquer avec le serveur EC2. Tout cela se fait très facilement.

S3 : Simple Storage Service. Service de stockage dans le Cloud. Il sert à héberger des fichiers tels que des images, programmes ou fichiers html. Il n’est pas un hébergeur de site web et n’est pas non plus utilisé comme un ftp. La plupart des entreprises s’en servent pour héberger leurs films (Netflix), images d’utilisateurs (airbnb), images de cours (Openclassrooms) ou encore les back-up des bases de données. Les avantages de S3 par rapport aux autres solutions, c’est la facilité de configuration des droits d’accès, la possibilité de versionner ses fichiers, de chiffrer les fichiers sensibles protégean ainsi les informations, de répliquer les fichiers entre différents datacenters ou encore d’indiquer une date d’expiration des fichiers afin de permettre à l’utilisateur de les supprimer.
